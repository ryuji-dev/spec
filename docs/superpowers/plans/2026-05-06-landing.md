## 작업 단계 (실제 진행 흐름)

### 1. CLAUDE.md 헌법 정합성 반영
- 기술 스택: "백엔드 미정" → PHP 8.x + MySQL on `recpc`. 프론트는 DB 직접 접근 금지, PHP API 경유.
- Next.js 버전 정책 명시: 16.x 외 사용 금지.
- 패키지 매니저 정책: pnpm. 명령어 예시도 `pnpm dev`/`pnpm build`/`pnpm add`로 갱신.
- 통신 규약 섹션 신설 (CORS Origin, OPTIONS 200, JWT Bearer, 응답 스키마).
- 보안 기본기 섹션 신설 (PDO Prepared, password_hash, 업로드 화이트리스트, 시크릿 분리, dangerouslySetInnerHTML 금지).
- TS strict 명시 (`tsconfig.json`은 이미 `strict: true`였음 — 검증만).
- 언어 정책 강화: 모든 사용자 응답·문서·주석·커밋 본문 한국어.
- 디자인 보존 규칙 강화: 마크업·Tailwind 클래스 임의 변경 금지.
- 디렉터리 트리에 `backend/` 형제 폴더 추가.
- 행동 규칙(헌법 [7]) 섹션 신설: 한 번에 너무 많은 파일 X, 추측 결정 X, 사전 1~2줄 요약.

### 4. 랜딩페이지 이식 (`frontend/src/app/page.tsx`)
**원칙**: 디자인 100% 보존. HTML→JSX 기계적 변환만. 폰트와 이미지 경로만 Next.js 표준으로 정상화.

- **레이아웃**: `<head>` 메타·폰트는 `frontend/src/app/layout.tsx`로, `<body>` 콘텐츠는 `page.tsx`로 분리.
- **폰트 self-host**: Google Fonts CDN 의존을 제거하고 `next/font/google`로 4종 로드 — Noto Serif KR, Noto Sans KR, Inter, Cormorant Garamond. CSS 변수(`--font-noto-serif-kr` 등)로 노출.
- **CSS Modules**: 디자인 원본 CSS를 `frontend/src/app/page.module.css`로 캡슐화. 글로벌 셀렉터(`html`, `body`, `*`, `button, a`)는 `globals.css`로 이전(CSS Modules는 글로벌 셀렉터 미지원).
- **이미지 경로**: `assets/background.jpg` → `/background.jpg` (`frontend/public/`로 평탄화).
- **인터랙션**: 원본의 `<script>` 안 함수는 모두 미사용이라 옮기지 않음. 메뉴와 ENTER는 `<a href="/main">`(메인페이지는 별도 PR이라 placeholder).
- **메타데이터**: title `서경노회 교육위원회 웹진`, description 한국어.
- **`html lang="ko"`** 적용.

### 5. 정중앙 정렬 보정 (사용자 1차 피드백)
첫 빌드 후 사용자 피드백:
- 중앙 섹션(`---EDUCATION COMMITTEE WEBZINE--- ~ 메뉴`)이 화면 정중앙이 아니라 위로 치우침
- 하단 SCROLL이 정중앙 아님

**원인**:
- `.content`가 flex `justify-content: space-between`인데 `.top`(작음)·`.bottom`(큼) 높이가 달라 `.center`가 시각적으로 위로 쏠림
- `.scrollHint`의 `margin: 0 auto` + flex가 `enter` 너비만큼 좌측으로 기울게 함

**해결**: 절대 위치 기반 레이아웃으로 재구성
- `.center`: `top:50%; left:50%; transform: translate(-50%,-50%)` 로 수학적 중심에 고정
- `.top`/`.bottom`: 화면 좌상/좌하에 절대 고정
- `.scrollHint`: `left:50%; transform: translateX(-50%)` 로 정중앙
- `.enter`: 우측 하단에 절대 위치
- 모바일 미디어쿼리(900px 이하)도 동일하게 조정

### 6. 스크롤 → 메인 진입 인터랙션 (사용자 1차 피드백)
사용자 요청: 마우스 휠을 내리면 ENTER 버튼을 누른 것과 같은 효과 (자동 라우팅).

**구현**: `frontend/src/app/scroll-to-enter.tsx` 클라이언트 컴포넌트(`'use client'`)
- `wheel` 이벤트로 `deltaY > 0` 감지 → `router.push("/main")` 한 번만
- 보너스: `touchstart`/`touchmove`로 위로 30px 이상 스와이프 시 동일 동작 (모바일 대응)
- `page.tsx`(서버 컴포넌트)는 그대로 두고, 이 작은 컴포넌트만 클라이언트화 — `'use client'` 최소화 원칙 준수

### 7. 배경 이미지 고화질 교체 (사용자 2차 피드백)
사용자 피드백: 배경 화질이 흐릿함. `~/Downloads/background.jpg` (4569×3046, 3.1MB)로 교체 요청.

**의사결정**: WebP 변환(옵션 C) 선택 — 해상도 유지, 파일 크기 절감.

**도구 시도**:
1. `sips -s format webp` → **Error 13: Can't write format: org.webmproject.webp** (macOS 25.4도 sips WebP 출력 미지원)
2. `cwebp`/`magick`/`ffmpeg` → 모두 미설치
3. **Python Pillow 11.3** 발견 → 사용

**결과**:
- Q85 method=6 → 2.4MB (목표보다 큼, 별·디테일 많아 압축 효율 낮음)
- **Q75 method=6 → 1.3MB** (3.1MB JPG 대비 58% 감소). 화질 대비 이 정도가 합리적이라 판단해 채택.
- `frontend/public/background.jpg` 삭제, `background.webp` 신규.
- `page.module.css`의 `url("/background.jpg")` → `url("/background.webp")` 한 줄 갱신.
- fallback(`image-set()`/`<picture>`) 미적용 — Caniuse 기준 WebP 지원 99%+, 후속 사용자 보고 시 추가.
