# 메인페이지 이식 — Phase 1 (정적 레이아웃 + mock 데이터)

> 본 plan은 `feat/main` 브랜치에서 실행되었으며, 대응하는 PR은 PR #2.
> 헌법 [작업 흐름]에 따라 plan 표준 경로(`docs/superpowers/plans/`)로 보관.

## Context

PR #1(랜딩페이지)이 머지된 직후, 랜딩의 `Enter`/메뉴 링크가 모두 `/main` 으로 이동한다 (`frontend/src/app/page.tsx:12`, `:53-58`, `:67`). 그러나 `/main` 라우트가 비어 있으면 404가 난다. 메인페이지 이식이 두 번째 큰 마일스톤이며, 디자인 원본 (`_design/.../서경노회 교육위원회 메인페이지.html` + `app.jsx` 1302줄 + `desktop.jsx` 357줄) 이 6개 섹션 × 데스크톱/모바일 분리로 구성되어 한 PR에 넣기엔 큼.

**이 PR(Phase 1)의 목표** — 헌법 [7] 준수:
- `/main` 라우트가 빈 페이지가 아니라 디자인 원본과 시각적으로 동일한 정적 레이아웃을 렌더하도록 만든다
- 공지·메뉴·일정·사진 데이터는 **mock 상수**로 시작 (백엔드 연동은 추후 별도 PR)
- 팔레트와 변형 옵션은 **기본값만**: `forest` / `kenburns` / `menu grid` / `photo feature`
- 디자인 원본의 마크업·여백·타이포·반응형은 100% 보존

**범위 외 — Phase 2 PR로 분리**:
- 히어로 자동 슬라이드(5.5초 루프) 및 켄번스 모션 — Phase 1은 슬라이드 #0 정지 화면
- 모바일 바텀탭 클릭 인터랙션 — Phase 1은 탭바를 렌더만 하고 active=0 고정
- 모바일 헤더 스크롤 효과 (`scrollTop > 80`)

## 트레이드오프 / 미해결

- **두 트리 동시 렌더**: 데스크톱·모바일 마크업이 모두 HTML로 내려가 첫 페이로드가 약간 커짐. 디자인 원본이 두 트리를 명시적으로 분리하기 때문에 채택. 후속 PR에서 lazy-load 또는 user-agent 기반 분기 검토 가능.
- **미디어 쿼리 임계값 (900px)**: 디자인 원본은 1440 / 402~412 두 지점만 디자인. 그 사이(태블릿 700~1100)의 어색함은 디자이너 추가 요청 대기.
- **메뉴 아이콘 색상**: 디자인 원본은 `palette` 객체를 prop으로 받아 동적으로 `fill`을 설정. CSS 변수 채택으로 forest 외 다른 팔레트로 toggle 시에도 자동 적용되지만, **SVG `fill="var(--palette-*)"` 가 일부 브라우저에서 미지원일 가능성** 있음. 시각 검증에서 확인 필요. 만약 문제가 있으면 컴포넌트로 currentColor 패턴 변환.
- **Phase 2 의존성**: 자동슬라이드를 추가할 때 `HeroMobile`/`DesktopHero` 가 클라이언트 컴포넌트로 승격됨. 슬라이드 SVG와 마크업 구조는 그대로 두고 `'use client'` + `useState(idx)` 만 도입하면 됨.
