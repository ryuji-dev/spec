# 신학원교수소개(/faculty) 페이지 이식

## 배경

글로벌 네비 항목 중 신학원교수소개(`main-page-data.ts`)가 라우트 미연결 상태였다. 디자인 핸드오프 번들(`서경노회 신학원교수소개.html` + `faculty.jsx` 1526줄)이 도착했고, 직전에 동일 흐름으로 완료된 `/resources` 이식과 같은 패턴으로 진행했다.

핸드오프 원본: `_design/seogyeong-presbytery-education-committee/`

## 결정 사항

- **라우트**: `/faculty` — 단일 영문 단어 컨벤션(`/webzine`·`/training`·`/board`·`/resources`) 준수. `MENU_ITEMS`의 `en: "FACULTY"`와 일치.
- **브랜치**: `feat/seminary-faculty` (분기 기준 `main`, `0626d2a`)
- **팔레트**: forest 단일 적용. `@/app/_components/shared/palette`의 `FOREST_PALETTE` 재사용. 원본의 olive/pine/sage 3종은 카드의 portrait/avatar tone 매핑에서만 사용(forest 기본 색계의 변주).
- **디바이스 분기**: `getDeviceType` 기반 SSR 분기 (desktop / ios / android).
- **컴포넌트 분리**: 데스크톱은 섹션별 파일, 모바일은 단일 파일.
- **Mock 데이터**: 원본 14명 교수 데이터의 **구조·필드는 그대로**, **한글/영문 이름은 가공의 가명으로 교체**(실제 교수진과 혼동 방지). 시간표·인용 스트립의 인명도 가명에 맞춰 일관 정리.

## 페이지 구조

데스크톱(1440px) 섹션 순서:
1. 글로벌 네비 — `<DesktopNav variant="solid" />`
2. `FeaturedHero` — 매거진 마스트헤드 + 커버 스토리 학장
3. `QuoteStrip` — 4명 교수 인용
4. `FilterStrip` — sticky 학과 필터 + Grid/List 토글
5. `FacultyGrid` / `FacultyList` — 3열 카드 또는 테이블 뷰
6. `ScheduleSection` — 시간표 표
7. `SeminaryCTA` — 신학원 안내
8. `FacultyFooter`

모바일: top bar → 마스트 → 커버 스토리 → 인용 스트립(가로 스크롤) → sticky 필터(가로 스크롤) → 카드 1열 스택 → bottom tab.

## 컴포넌트 재사용 매핑

### 전역 import (재사용)
- `FOREST_PALETTE`, `Palette` 타입 (`@/app/_components/shared/palette`)
- `<DesktopNav variant="solid" />` (글로벌 헤더)
- `getDeviceType` (`@/lib/device`)

### 라우트 로컬 신규 작성
- `_components/deptTone.ts` — `deptColor` / `avatarTone` / `portraitTone` (학과·tone 색상 매핑)
- `_components/shared/FacAvatar.tsx` — 작은 아바타 원형
- `_components/shared/FacultyPortrait.tsx` — 학과·tone 별 procedural SVG 초상화
- `_components/desktop/{FeaturedHero, QuoteStrip, FilterStrip, FacCard, FacultyGrid, FacultyList, ScheduleSection, SeminaryCTA, FacultyFooter}.tsx`
- `_components/mobile/FacultyMobile.tsx` — 모바일 단일 파일

### 의식적으로 재사용 안 함
- 글로벌 `BottomTabBar` — 디자인 원본 모바일 bottom tab 이 점 인디케이터 + ko/en 듀얼 라벨로 별도 변형이라 인라인 보존.
- 다른 페이지의 카드 칩(LbCatLabel 등) — 학과 배지는 형태가 달라 카드 내부에 인라인.

## 디자인 보존 규칙

- 마크업·인라인 스타일·색상·타이포·여백 변경 금지. 데이터 바인딩·라우팅·상태만 추가.
- HTML→JSX 기계적 변환만 (`class`→`className`, self-closing, 인라인 `style` 객체화).
- 외부 폰트(Noto Serif KR / Noto Sans KR / Inter)는 글로벌 layout 의 `next/font` self-host 그대로.
- `dangerouslySetInnerHTML` 금지.
